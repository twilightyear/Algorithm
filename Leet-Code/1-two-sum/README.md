# [LeetCode] 1. Two Sum


---


### 📌 문제 링크
[LeetCode - Two Sum](https://leetcode.com/problems/two-sum/)


---

### ⚠️ 제약조건
- 2 <= nums.length <= 10^4
- -10^9 <= nums[i] <= 10^9
- -10^9 <= target <= 10^9
- Only one valid answer exists.



---


### 🛠️ 풀이 접근 및 핵심 로직
1. **2차원 배열 생성** : 2차원 배열을 만들고 [기존값,기존인덱스] 를 모든 요소에 있어서 만든다. -> 시간복잡도 O(n)
2. **Sort** : sort를 사용하여 기존값 기준으로 정렬한다. -> 시간복잡도 O(nlogn)
3. **Two Pointer** : 이 문제는 답이 한쌍만 존재한다고 가정했다. two pointer 알고리즘을 사용하여 target 값을 구하는 정답의 인덱스인 left과 right을 증감을 통하여 구하고 반환한다. -> 시간복잡도 O(n)


---


### 📊 복잡도 분석
**시간복잡도**

시간복잡도는 총 O(nlogn)이다.
데이터가 10^7을 넘어가면 O(nlogn)으로도 시간초과가 날 수도 있다. 하지만 nums의 길이는 최대 10^4까지만 나온다고 한정이 되어있으니 충분하다.

**공간복잡도**

기존 배열값에서 기존 인덱스값까지 가진 리스트를 새로 할당했다. 그럼으로 O(n)의 공간복잡도를 가진다. 문제에서 공간복잡도를 제한하지 않았으니 문제는 없다.



---


### 📝 회고 / 추후 개선점
문제를 풀고 나서 다른 사람들의 풀이와 내 풀이를 비교해봤다. Hashmap을 사용하는 방법이 있으며, O(n)으로 시간복잡도와 공간복잡도 개선을 할 수 있다는 사실을 파악했다. 추후에 다시 한번 풀어보겠다.
