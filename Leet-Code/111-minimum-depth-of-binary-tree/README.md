# [LeetCode] 111. Minimum Depth of Binary Tree


---


### 📌 문제 링크
[LeetCode - Minimum Depth of Binary Tree](https://leetcode.com/problems/minimum-depth-of-binary-tree/description/?envType=problem-list-v2&envId=depth-first-search)


---

### ⚠️ 제약조건

- The number of nodes in the tree is in the range [0, 105].
- -1000 <= Node.val <= 1000


---


### 🛠️ 풀이 접근 및 핵심 로직
- 본 문제는 이전 풀어봤던 최대 길이를 구하는 DFS 유형과는 다르게 가장 짧은 길이를 찿아내는 문제다. 여기서 이전 유형처럼 max() 대신 min()를 사용하여 재귀를 사용한다면 자식이 없는 부분에 0으로 처리해야하는 상황에서 항상 0만 반환하여 결국 문제가 이상하게 풀린다. 그렇기에 자식이 2개 전부 존재하는 상황과 아닌 상황을 분리하여 문제에 접근한다. 2개 전부 존재하는 상황에서는 위에서 설명했던 문제는 발생하지 않는다. 그렇기에 min(left,right)+1 를 사용하여 자식 노드 중에서 작은 깊이를 가지는 것을 선택하고 그 깊이에 1을 더하여 반환한다. 하지만 하나라도 존재하지 않는다면 여기서 상황을 더 봐야한다. 만약 자식이 전부 없다면 0을 반환하면 되며, 하나만 존재하는 상황이라면 그 하나가 반환하는 노드값을 가져오면 된다(그 하나가 만약 리프노드라면 0을 반환할것이다) 하나만 존재하는 상황이라면 left가 존재하는 노드인지 right가 존재하는 노드인지 별도의 구분과 선택이 필요하다. 조건문의 사용을 하지 않고 단순하게 left+right를 사용하여 값의 도출이 가능하다. 이 수식은 적어도 하나의 자식 노드는 존재하지 않는다라는 가정 하에 참이다. 이 left+right 를 사용하여 선택이라는 논리를 구현하고 반환하는 과정에서 +1를 더하여 깊이의 증가를 해준다. 이렇게 상황의 분리를 통하여 문제를 해결할 수 있었다.
---


### 📊 복잡도 분석

**시간복잡도**
- 이진 트리라는 조건에서 모든 노드의 조회를 진행한다. 중복되게 조회되는 경우는 없다. 그럼으로 O(N)의 시간복잡도를 가질것이다.

**공간복잡도**
- 이진 트리라는 조건에서 최악의 경우 편향트리의 조회를 진행한다. 이럴 경우 노드 개수 N개에 대하여 N개의 재귀 스택이 필요하다. 그럼으로 O(N)의 공간복잡도를 가질것이다.

### 📝 회고 / 추후 개선점
- 단순하게 **max**를 사용하는 문제를 풀다가 본 문제에서 최소를 막연하게 풀려고 하다보니 시행착오가 생각보다 있었던 문제였다. 아무리 생각해도 문제해결의 단서를 찿을수가 없어서 다른 사람들의 풀이를 읽어보고 예재와 그림을 그려보며 이해하려는 노력 끝에 원리와 의도를 이해할 수 있었다.
