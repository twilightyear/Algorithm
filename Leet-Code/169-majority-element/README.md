# [LeetCode] 169. Majority Element
---
### 📌 문제 링크
[LeetCode - 169. Majority Element](https://leetcode.com/problems/majority-element/description/?envType=problem-list-v2&envId=hash-table)

---

### ⚠️ 제약조건
- n == nums.length
- 1 <= n <= 5 * 104
- -109 <= nums[i] <= 109
- The input is generated such that a majority element will exist in the array.
---
### 🛠️ 풀이 접근 및 핵심 로직
- 본 문제를 보자마자 두가지 가능한 풀이 방법을 구상했다. 첫번째는 **Hashmap**의 이용이다. 모든 값들을 순회하면서 **Hashmap**을 이용하여 개수를 **Count** 하고 **max** 값을 구한다면 쉽게 최빈값의 도출이 가능할 것이다.
- 하지만 문제 조건에서 **공간복잡도** **O(1)**로 풀어봐라고 도발을 해온다. 이건 쉽게 넘어갈 수 없다. 아쉽게도 **Hashmap**을 이용한다면 최악의 경우, 최빈값이 존재하기 위한 2개의 값만 동일하고 나머지 요소의 개수 N개에 대하여 **O(N)**의 공간복잡도를 가진다.
- 그렇기에 처음 떠올린 두번째 방법의 적용이 필요하다. 바로 정렬의 방법이다. 정렬을 사용한다면 요소의 개수와는 무관하게 결국 중앙의 값은 최빈값의 차지가 된다.

---
### 📊 복잡도 분석
**시간복잡도**

- **Python** 에서 기본 내장 **Sort **는 **Tim Sort** 를 사용한다. **Tim Sort**는 **O(NlogN)**의 시간복잡도를 가진다.

**공간복잡도**

- **Hashmap** 방식과는 다르게 최소한의 공간이 사용된다. **O(1)**의 공간복잡도를 가진다.

### 📝 회고 / 추후 개선점
- **Hashmap** 과 **Sort** 의 방식을 고르게 고려하여 더욱 적절한 풀이방법을 택할 수 있었다. 이러한 풀이 양상이 더욱 어려운 문제로 가서도 동일하게 적용되어 건설적인 공부와 풀이가 될 수 있는 실력을 길러야겠다. 또한, 현재 군대에서 제한적으로 문제를 계속해서 풀어나가고 있다. **DFS** 위주로 문제를 풀고 있는데, 문제 풀이 이외에도 조금 더 지엽적으로 학습적인 내용 또한 심도있게 공부하여 깊이있는 이해를 통한 심화적인 문제에 더욱 적극적으로 접근해보고 싶다.
