# [LeetCode] 5. Longest Palindromic Substring


---


### 📌 문제 링크
[LeetCode - Longest Palindromic Substring](https://leetcode.com/problems/longest-palindromic-substring/description/)


---

### ⚠️ 제약조건
- 1 <= s.length <= 1000
- s consist of only digits and English letters.



---


### 🛠️ 풀이 접근 및 핵심 로직
- 핵심은 중심 기준 확장이다. 중심이란 말 그대로의 글자 중심이 아니라 처음부터 문자 하나 하나 순회를 하지만 그것을 중심으로 확장한다는 의미다. 다만 팰린드롬이 aba와 같은 홀수 팰린드롬일 수도 있으며 abba와 같은 짝수 팰린드롬일 수 있다는 사실을 고려하여 별도로 구해주고 고려해줘야 한다. 순회를 계속 하며 max 내장 함수를 key=len와 함께 사용하여 최대길이값에 따른 갱신을 계속 해주면 결국 최종 정답을 구할 수 있다.

### 📊 복잡도 분석
**시간복잡도**
- 본 문제는 for 문을 통하여 글자 하나하나 순회를 하는것으로 시작한다. 단어의 길이인 L에 대하여 O(L)의 시간복잡도를 가진다. 여기서 중첩으로 투포인터를 이용한 슬라이딩 윈도우 알고리즘으로 확장 및 확인의 논리를 가지고 있는데, 최악의 경우 전부 순회해야하니 O(L)의 시간복잡도를 가질것이다. 그럼으로 O(L^2)의 시간복잡도를 가질것이다.

**공간복잡도**
- 추가적인 공간사용은 있지만 결국 left right 포인터를 사용하며 odd 와 even도 결국 유사하다. even과 odd도 최대로 결국 단어의 길이인 L에 대하여 O(L)의 공간복잡도를 가질 것이다. 결론적으로 O(L)의 공간복잡도를 가진다.

### 📝 회고 / 추후 개선점
처음에는 단순하게 슬라이딩 윈도우만 적용해서 간단하게 구현하려고 했으나 구현하고 나서 시간복잡도가 끔찍할 정도로 높다는 것을 파악하고 중심 확장 방법으로 방향을 틀었다. 구현에 너무 오랜 시간이 걸려서 다른 사람들의 풀이를 참고해보니 본 문제에서 함수 안에 함수를 넣는 것으로 쉽게 중심 기준 확장을 쉽게 할 수 있다는 아이디어를 얻고 다시 문제를 풀어보았으며 이렇게도 풀 수 있겠구나 라는 것을 배울 수 있었다.
