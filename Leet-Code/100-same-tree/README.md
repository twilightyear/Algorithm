# [LeetCode] 100. Same Tree

---


### 📌 문제 링크
[LeetCode - 100. Same Tree](https://leetcode.com/problems/same-tree/description/?envType=problem-list-v2&envId=depth-first-search)


---

### ⚠️ 제약조건
- The number of nodes in both trees is in the range [0, 100].
- -104 <= Node.val <= 104



---


### 🛠️ 풀이 접근 및 핵심 로직
- 내부에 함수를 선언하여 재귀적으로 푸는 방식을 선택했다. 왼쪽 서브트리, 오른쪽 서브트리로 각각 나눈 후 DFS 방식으로 구현했다. 본 코드에서는 Preorder 방식으로 조회하는 방식으로 시작된다. 
조회방식이 구현되었으니 비교는 간단하다. 한번의 조회에서 두 트리를 한번에 움직이게 하면 될 것이다. 먼저 두 트리의 노드가 둘다 None이면 비교할 필요가 없음으로 True를 반환한다. 위의 조건이 아닌 경우에서 하나가 None이라면 하나만 None이라는 말이다. 그럼으로 False를 반환한다. 위의 조건에 걸리지 않았다고 해도 두 노드의 값이 다르다면 False를 반환한다.
- 이렇게 재귀적으로 조회된 왼쪽 서브트리와 오른쪽 서브트리의 값 둘다 True가 나와야 진정한 True로 반환할 수 있음으로 and 문을 사용하여 최종 반환을 해주면 결과를 도출할 수 있겠다.

---


### 📊 복잡도 분석
**시간복잡도**
- 본 문제에서는 탐색 과정에서 노드를 비교하는 방식으로 동작한다. 각 노드당 최대 1번만 방문하게 되어있음으로, 시간복잡도는 O(n)이라고 할 수 있다.

**공간복잡도**
- 재귀적으로 탐색을 진행하며 추가적인 리스트나 트리의 할당은 하고있지 않다. 이러한 측면에서는 O(1)의 공간복잡도를 가진다. 하지만 트리구조의 재귀적인 순회를 위해서는 호출스택이 사용되는데 호출 스택인 트리의 높이 h에 대하여 O(h)의 공간복잡도를 가진다. 만약 최악의 경우로 생각해볼수 있는 한쪽에만 쏠려있는 트리인 편향트리가 있다고 가정해보면 편향트리가 가지는 노드의 수 n 에 대하여 O(n)의 공간복잡도를 가질 수 있다.

### 📝 회고 / 추후 개선점
- 처음 문제를 보자마자 Trie 구조를 쓰는건 어떤가 하는 생각이 들었지만 굳이 무겁게 Trie 클래스를 적재하면서까지 필요가 없을 것이라는 생각이 들었으며, 본 문제 특성상 값뿐만 아니라 위치도 중요하니 적용할 수 없는 문제라고 판단했다. 이후에는 DFS, BFS 로 적용하는게 어떤가라는 생각을 해보았다. 본 문제에는 솔직히 어떤 방식이 적용되어도 결과적으로 아무 문제가 없을 것이라는 생각을 했으나, 현재 DFS 문제들을 집중적으로 연습하고 있기도 하고, 조금이라도 직관적인 풀이를 위하여 DFS를 선택하여 풀어보았다.
