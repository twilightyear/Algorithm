# [LeetCode] 20. Valid Parentheses


---


### 📌 문제 링크
[LeetCode - 20. Valid Parentheses](https://leetcode.com/problems/valid-parentheses/)


---

### ⚠️ 제약조건
- 1 <= s.length <= 104
- s consists of parentheses only '()[]{}'.



---


### 🛠️ 풀이 접근 및 핵심 로직
- 먼저 이 문제는 stack를 하나 만들어서 활용하는것이 핵심 포인트이다. 괄호의 종류와 대응되는 값들은 dictionary를 이용했다.  먼저 들어오는 괄호들에 있어서 순서대로 접근한다. 여기서 역으로 보든 순방향으로 보든 결국 valid하나 valid하지 않나는 동일하다. 먼저 값이 dictionary에 key로 존재하는지 확인한다. 여기서 dictionary 조회상 시간/공간복잡도 측면에서 우수한 성능을 보일것이다. 이렇게 key가 존재한다는 사실은 순방향으로 조회하는 로직 구조상 닫는 기호로 끝난다는 것인데 만약 처음에 이러한 값이 나왔다는 것은 뒤를 보지 않고도 valid 하지 않다라는 것을 확인할 수 있다. 그렇기에 stack이 비어있다는 정보만 확인하고 바로 False를 반환하면 되겠다. 만약 stack에 값이 무엇이라도 있다라고 하면 말이 조금 달라진다. 처음 값이 아닌 이상 앞에 무엇이 있었느냐에 따라 valid 여부가 갈린다. dictionary에 설정한 값에 따라 여는 기호가 들어올 경우 stack에 추가하는데, 만약 stack을 pop 했을때 나온 값과 위에서 말한 값이 동일한 경우에는 문제가 없다고 할 수 있겠지만 다른 경우는 뒤를 더 확인할 필요도 없이 valid하지 않다라는 사실을 도출할 수 있다. 
- 이렇게 False 반환 없이 처리가 완료되었을때 stack에 아무것도 남아있지 않았다면 vaild하다라는 결론을 도출 할 수 있는 것이다.

### 📊 복잡도 분석
**시간복잡도**
- for 문을 통하여 s의 길이 L에 대하여 O(L)의 시간복잡도를 가진다. 2가지 경우에 있어서 중도에 False로 끝내버리는 경우도 있지만 최악의 경우를 가정한다. 이외에 여는 부호의 종류가 나올 경우 추가를 해주는 과정이 존재하는데, 최악의 경우 길이 L에 대하여 O(L)의 시간복잡도를 가질 것이다. 마지막으로 return 하기 전에 stack의 길이를 확인하는 작업이 있는데, 마찬가지로 최악의 경우 길이 L에 대하여 O(L)의 시간복잡도를 가진다.

- 종합적으로 O(L)의 시간복잡도를 가진다라는 것을 분석해볼 수 있었다.

**공간복잡도**
- 핵심적인 로직을 담당하는 stack은 최악의 경우 길이 L에 대하여 O(L)의 공간복잡도를 가진다. 빠른 조회를 위한 dictionary는 최초 정의 이외에 따로 공간을 추가적으로 더 사용하지 않음으로 O(1)의 공간복잡도를 가진다. 
- 종합적으로 O(L)의 공간복잡도를 가진다라는 것을 분석해볼 수 있었다.
