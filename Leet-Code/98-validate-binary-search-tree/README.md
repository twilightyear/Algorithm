# [LeetCode] 98. Validate Binary Search Tree


---


### 📌 문제 링크
[LeetCode - Template](https://leetcode.com/problems/validate-binary-search-tree/description/?envType=problem-list-v2&envId=depth-first-search)


---

### ⚠️ 제약조건
- The number of nodes in the tree is in the range [1, 104].
- -231 <= Node.val <= 231 - 1



---


### 🛠️ 풀이 접근 및 핵심 로직
- 단순한 **DFS** 조회로는 자식노드 방향으로의 대소유무의 전체 조회에는 어려움이 있을것이다. 그렇기에 함수를 따로 선언하여, **-inf** 와 **+inf** 를 이용한 **high** 값과 **low** 값을 설정하여 범위에 들어가는지 확인하고, 오른쪽 노드에는 **low < 왼쪽노드 < 현재노드값** 을 탐색하도록 하는 재귀를 실행하고, 왼쪽 노드에는 **현재노드값 < 오른쪽노드 < high** 를 탐색하도록 하는 재귀를 실행하도록 한다. 쉽게 말하자면, 루트 노드가 있다고 예시를 들자. 루트 노드가 값을 **a**를 가진다고 하면 루트 기준 왼쪽 노드는 **-inf** 부터 **a**까지의 값만 가질 수 있으며, 오른쪽 노드는 **a** 부터 **inf**까지의 값만 가질수 있다. 이러한 논리는 계속 아래로 가도 적용된다고 가정하면 직계자식까지만 조건이 적용되는 문제를 쉽게 해결할 수 있다.
---


### 📊 복잡도 분석
**시간복잡도**
- 노드 개수 **N**개에 대하여 결국 전부 조회를 필요로 한다. 그렇기에 **O(N)**의 시간복잡도를 가진다.

**공간복잡도**
- **Binary Tree** 인지 유무를 조회하는 코드이며, 결국 최악의 경우 길이가 **N** 인 편향트리라는 가정 하에 재귀스택을 호출을 해야함으로 **O(N)** 의 공간복잡도를 가지게 된다.




---


### 📝 회고 / 추후 개선점
- 솔직히 말하면 문제를 접하고 정말 쉽게 풀 수 있을 줄 알았지만 생각보다 구현에 애를 먹었다. 예를들자면, 왼쪽 노드의 값은 현재 노드의 값보다 작아야한다. 이러한 특성을 구현하기 위해 단순하게 왼쪽 노드의 값은 현재 노드의 값보다 작도록 재귀문을 통하여 오른쪽과 함께 구현했으나 문제가 발생했다. 단순하게 자식에서 한정된 것이 아닌 자손들 전부 조건을 만족해야 한다는 점의 문제였다. 트리의 예시를 들어보겠다.

```
5
└── 1 └── 7
               ├── 3
               └── 8
```

- 여기서 위에서 말했던 논리대로라면 **True**를 반환한다. 

- 하지만 **Binary Tree** 가 아니라는 것을 알 수 있을 것이다. 

- 이러하게 직계자식만 보는 문제를 해결하기 위하여 **-inf** 와 **+inf** 를 활용한 **low** 및 **high** 를 통한 재귀 및 범위확인을 이용하면 문제를 해결할 수 있다는 사실을 알게 되었다.
