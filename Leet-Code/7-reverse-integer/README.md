# [LeetCode] 7. Reverse Integer


---


### 📌 문제 링크
[LeetCode - Reverse Integer](https://leetcode.com/problems/reverse-integer/)


---

### ⚠️ 제약조건
- -2^31 <= x <= 2^31 - 1
---


### 🛠️ 풀이 접근 및 핵심 로직
1. **절댓값 변환** : 부호를 기억하고 절댓값으로 변환한다.
2. **Reverse** : 숫자를 뒤집고 부호를 이후에 붙여준다.
3. **결과 범위 확인** : 문제에서 제시한 범위 안에 결과가 있는지 확인한다.



---


### 📊 복잡도 분석
**시간복잡도**

정수를 문자열로 만들고 뒤집으며 이후 다시 정수로 만드는 과정은 전체적으로 O(n)의 시간복잡도를 지닌다. 부호를 구하는것과 이후에 문제에서 제시한 범위인지 확인하는 일련의 과정도 O(1)의 시간복잡도를 가진다.
전체적인 시간복잡도는 그럼으로 O(n)이라고 할 수 있다.

**공간복잡도**

부호를 기억하는것은 O(1)의 공간복잡도를 가지며, reverse를 수행하는과정에서 발생하는 공간의 사용은 O(n)의 공간복잡도를 가진다.
전체적인 공간복잡도는 O(n)이라고 할 수 있다.


---


### 📝 회고 / 추후 개선점
본 문제를 풀때 Python의 [::-1]를 적극 사용하여 문자열을 뒤집는 방식으로 해결을 했다. 다만 문자열을 뒤집는 것이 아니라 수를 이용한 계산으로 풀 수 있었다면 시간단축의 가능성이 있는 것이 아닌가라는 생각이 들었다. 또한 2의 31승과 같은 정보를 프로그램이 직접 계산하는데, 미리 우리가 알고 있는 값이며 변화의 여지가 없는 값이니 사전에 계산된 값을 적어두는 것도 사소하지만 시간단축에 의미가 있지 않을까라는 생각을 해봤다. 추후에 개선된 버전을 만들어서 시간단축을 할 수 있는지 시험해보겠다.
