# [LeetCode] 101. Symmetric Tree


---


### 📌 문제 링크
[LeetCode - Symmetric Tree](https://leetcode.com/problems/symmetric-tree/description/?envType=problem-list-v2&envId=depth-first-search)


---

### ⚠️ 제약조건

- The number of nodes in the tree is in the range [1, 1000].
- -100 <= Node.val <= 100


---


### 🛠️ 풀이 접근 및 핵심 로직
- 본 문제는 Tree가 대칭되는 요소를 가지고 있는지 확인하는데 있다. 문제를 보자마자 생각이 들었던 것이 Preorder 순과 Postorder 순의 탐색이였다. 이를 활용하면 조회 로직 2개만 구현해도 편하게 대칭 여부를 파악할 수 있을 것이다.


---


### 📊 복잡도 분석
**시간복잡도**
- 트리가 편향트리인 최악의 경우, 길이 N에 대하여 O(N)의 시간복잡도를 가진다. 이외에도 최종 배열 비교 과정에서 총 N번의 조회를 마쳐야 함으로 O(N)의 시간복잡도를 가진다. 종합적으로 결국 O(N)의 시간복잡도를 가진다고 할 수 있겠다.

**공간복잡도**
- 본 조회의 특성상 최종 비교를 위하여 배열에 따로 저장을 하고 있다. 그렇기에 N개의 요소들을 저장하기 위한 총 N개의 공간이 필요한데, 그렇기에 O(N)의 공간복잡도를 가진다고 할 수 있다.




---


### 📝 회고 / 추후 개선점
- 구현하기 위해 떠오른 방법을 무작정 적용하여 풀어보았다. 하지만 풀어보고 다른 사람들의 풀이와 비교해보니 다소 비효율적이라는 평가를 했다. 이유는 조회와 비교 로직에 있다. 접근법은 나쁘지 않았으나 굳이 배열에 따로 저장하고 이후에 다시 비교할 이유가 크게 없었던것이다. 이외에도 재귀의 방식을 사용하면 간결하면서도 더 빠른 동작을 하는 코드를 구현할 수 있었겠다라는 고찰이 있었다.
