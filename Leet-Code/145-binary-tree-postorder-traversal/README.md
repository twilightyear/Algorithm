### ⚠️ 제약조건
- The number of the nodes in the tree is in the range [0, 100].
- -100 <= Node.val <= 100
---
### 🛠️ 풀이 접근 및 핵심 로직
- 이전에 풀었던 **144번 문제** 와 유사한 유형의 문제이다. 다만 **Preorder** 순이 아닌 **Postorder** 순으로 **traversal** 해야한다. 재귀적인 구성에서 **Postorder** 순으로 조회할 수 있도록 코드 위치적 구성을 신경쓰면 해결이 가능하다.
---
### 📊 복잡도 분석
**시간복잡도**

- **N** 개의 노드에 있어서 1번씩의 조회를 무조건 하게 된다. 그렇기에 **O(N)**의 시간복잡도를 가진다.

- **result** 리스트에 노드 개수만큼 결국 값이 추가가 되는데, 전체적으로 보아도 결국 **O(N)**의 시간복잡도를 가진다.

- 최고차항에 따라 전체적인 시간복잡도는 **O(N)** 이라고 할 수 있다.

**공간복잡도**

- 이번 문제에서도 역시나, 편향트리라는 최악의 경우에 있어서 노드 개수가 **N** 개인 경우에서 고려가 필요하다. 재귀적인 구조로 작동하는 본 코드에서는 재귀호출스택이 사용되는데 앞의 최악의 경우에서 **O(N)**의 공간복잡도를 가진다.
- 이외에도 결과 반환용으로 사용한 **result** 리스트에 노드 개수 **N** 만큼의 개수가 추가가 된다. 그렇기에 **O(N)** 의 공간복잡도를 가진다.
- 최고 차항에 따라 전체적인 공간복잡도는 **O(N)**이라고 할 수 있다.
